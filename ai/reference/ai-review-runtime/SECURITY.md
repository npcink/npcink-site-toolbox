# 安全说明

> **禁止直接投产。** 快照忠实保留历史实现，因此也保留已知安全和可靠性缺陷。SHA-256 只证明内容完整，不代表代码安全。

## 已知高风险

| 风险 | 原实现 | 影响 | 移植前要求 |
| --- | --- | --- | --- |
| 同步阻塞 | 在 `preprocess_comment` 内执行最长 15 秒的外部请求 | Provider 变慢会拖慢或耗尽评论请求 | 优先异步待审；同步时使用短预算、限流和断路器 |
| 异常时放行 | 网络错误、非 200、JSON 解析错误多次返回 `is_safe=true` | 审核服务故障会被误判为内容安全 | 使用独立错误状态；按显式 Policy 进入待审或阻止 |
| 自定义 URL SSRF | 管理员可配置任意 URL、Method、Header 和 Body | 可探测私网/云元数据，或把凭据发送到错误目标 | 固定 Provider 或严格 Host allowlist，并校验解析地址与重定向 |
| 配置与密钥回显 | 原管理表单使用完整 `ai_review` 配置对象 | API Key、Secret、自定义 Header 可能进入浏览器和设置响应 | 浏览器只得配置状态/掩码；密钥只在服务端 |
| 不稳定日志标识 | REST 用数组下标审核日志，前端根据页码计算位置 | 并发插入新日志后可能操作错误记录 | 使用不可变 UUID/数据库主键和版本检查 |

## 其他必须评估的问题

### Provider 响应校验

- DeepSeek 实现用宽松正则提取第一个 JSON 对象，未完整验证字段类型和范围。
- 自定义 API 把缺失 `is_safe` 当作安全，把缺失风险等级当作 `safe`。
- 阿里云端点、签名协议和响应结构属于易变化的外部契约，移植时必须重新查证当前官方文档。
- 原因文本只做基础清洗；目标系统仍应在每个输出上下文转义。

任何响应缺字段、类型错误、超范围、超大或无法解析时，都应进入 `invalid_response`，而不是安全通过。

### 本地正则

管理员提供的正则通过 `@preg_match` 静默执行：

- 错误规则没有可见反馈；
- 灾难性回溯可能消耗 CPU；
- 匹配逻辑把任何返回真值都视为命中，但没有规则级超时和复杂度约束。

目标实现应在保存时编译验证规则、报告错误、限制长度/数量，并评估是否只允许受控规则格式。

### 日志与隐私

历史日志包含评论者姓名、邮箱、正文片段、判定和时间，并把整个数组存入单一 Option：

- 数据会随着条目增加反复整块读写；
- 邮箱和评论属于个人信息；
- 清空是全量删除，缺少逐条保留期与可审计删除；
- Provider 错误原因可能泄露服务端信息。

移植时必须最小化字段、设置明确保留期、限制查看权限、提供删除路径，并避免记录密钥、Authorization Header、完整 Provider 响应或不必要的评论全文。

### REST 与人工动作

原端点有 `manage_options` 检查，但目标插件仍必须单独验证：

- Cookie/nonce 鉴权；
- 每个写操作的 capability；
- 参数枚举和 Schema；
- 重放与幂等性；
- 审核记录版本未在操作前变化；
- 错误响应不携带密钥或敏感供应商详情。

## 凭据与归档纪律

本参考目录只能保存源代码中的占位值，例如 `sk-...`、`LTAI...`、`Bearer xxx`。不得加入：

- 真实 API Key、AccessKey ID、Secret；
- 真实 Authorization/Cookie/Header；
- WordPress 设置导出、数据库备份或日志导出；
- `.env`、调试转储、抓包内容或生产域名私有端点。

移植前后都应扫描常见密钥格式、高熵字符串和私钥头，并人工复核命中。示例占位符命中不能简单全部忽略，应确认它们确实来自文档或 UI placeholder。

## 生产前阻断条件

以下条件任一未满足，就不能启用外部 Provider：

1. 密钥完全保持在服务器端，GET 响应和页面源代码不可见。
2. SSRF、重定向和 DNS 解析边界有自动化测试。
3. 超时、网络错误、限流和解析错误有独立状态与显式处置策略。
4. 评论主请求不会被 15 秒外部调用阻塞。
5. 日志使用稳定 ID，并有权限、隐私、保留和删除策略。
6. Provider 当前契约已按官方文档重新实现并有契约测试。
7. 发布 ZIP 不包含本参考目录、测试凭据或设置导出。
