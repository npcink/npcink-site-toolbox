# WP Magick Toolbox — 按需加载架构规范

> 版本：1.0 | 生效日期：2026-05-08 | 适用版本：2.0.84+

> [!WARNING]
> 本文件保留早期按需加载设计背景，其中的前端目录和分包示例已被 Pre-GA 重构取代，不再作为当前实现指引。当前前端与发布边界见根目录 `README.md`、`docs/构建与发布指南.md` 与 `docs/adr-pre-ga-admin-reset-2026-07-15.md`；模块加载事实以 Registry、Loader 和测试为准。

## 一、背景与目标

### 1.1 现状问题

插件当前拥有 **90+ 项功能**，但加载架构存在以下问题：

| 问题 | 描述 | 影响 |
|---|---|---|
| **无条件加载** | `MaBox_Admin::load()` 无条件加载 7 大模块及其所有子模块 | 即使用户只开启 3 个功能，所有 PHP 文件仍被解析到内存 |
| **磁盘 I/O 浪费** | 每个 `require_once` 都产生文件读取开销 | 90+ 文件的 I/O 累积拖慢 `init` 钩子响应 |
| **内存占用高** | 所有类的定义、Hook 注册全部执行 | 每个 WP 请求多占用 5-10MB 内存 |
| **Hook 爆炸** | 未启用功能的 Hook 也被注册 | 增加 WP 钩子调度开销 |

### 1.2 目标

| 指标 | 当前 | 目标 |
|---|---|---|
| 单请求加载 PHP 文件数 | 90+（全部） | ≤ N（N = 已开启功能数） |
| 关闭功能的内存占用 | ~5-10MB | **0**（零开销） |
| 前端 HTTP 请求数（开 20 功能） | 20+ | ≤ 8 |
| 后台首屏加载时间 | ~2s | ≤ 0.8s |

## 二、核心架构设计

### 2.1 加载流程对比

#### 改造前（当前架构）

```
magick-tool-box.php
  └── Magick_Mixtrue::run()
        └── MaBox_Admin::load()
              ├── require optimize/index.php      ← 无论是否开启任何优化功能
              │     └── MaBox_Optimize::run()
              │           ├── require site/index.php
              │           ├── require medium/index.php
              │           └── require admin/index.php
              ├── require function/index.php       ← 同上
              ├── require h5.php                   ← 同上
              ├── require login/index.php          ← 同上
              ├── require page/index.php           ← 同上
              ├── require shortcode/index.php      ← 同上
              └── require template/index.php       ← 同上
```

#### 改造后（按需加载架构）

```
magick-tool-box.php
  └── Magick_Mixtrue::run()
        └── MaBox_Admin::load()
              ├── 读取 Magick_ToolBox_Active_Modules（一维数组）
              └── foreach $active_module as $path
                    ├── require $path              ← 仅加载已开启的功能
                    └── ModuleClass::activate()    ← 统一激活入口
```

## 三、PHP 模块开发规范

### 3.1 模块文件结构

每个功能模块必须遵循以下结构：

```php
<?php
/**
 * 功能名称：XXX
 * 所属类别：页面外观 / 站点优化 / ...
 * 依赖：无 / MaBox_Helpers / ...
 */

if (!class_exists('Npcink_Page_XXX')) {
    class Npcink_Page_XXX
    {
        /**
         * @var array 模块配置
         */
        private static $option;

        /**
         * 激活模块（统一入口）
         * 
         * @param array $config 该功能的配置数据
         */
        public static function activate($config)
        {
            self::$option = $config;
            
            // 注册 WordPress Hooks
            add_action('wp_footer', array(__CLASS__, 'render'));
            add_filter('the_content', array(__CLASS__, 'filter_content'));
        }

        /**
         * 渲染输出
         */
        public static function render()
        {
            $enabled = MaBox_Admin::get_config(self::$option, 'feature_key');
            if ($enabled !== true) {
                return;
            }
            
            // 功能逻辑
            echo '<div>...</div>';
        }

        /**
         * 内容过滤
         */
        public static function filter_content($content)
        {
            $enabled = MaBox_Admin::get_config(self::$option, 'feature_key');
            if ($enabled !== true) {
                return $content;
            }
            
            // 过滤逻辑
            return $content;
        }
    }
}
```

### 3.2 命名规范

| 元素 | 规范 | 示例 |
|---|---|---|
| **类名** | `Npcink_` 或 `MaBox_` 前缀 + 功能描述（PascalCase） | `Npcink_Page_Copy_Pop_Up` |
| **文件名** | 小写下划线，与类名对应 | `copy_pop_up.php` |
| **配置键** | 小写下划线，语义清晰 | `copy_pop_up` |
| **Hook 回调** | 静态方法，语义明确 | `render()`, `filter_content()` |

### 3.3 模块独立性要求

**每个模块必须是完全独立的闭环**，禁止以下行为：

| 禁止行为 | 原因 | 正确做法 |
|---|---|---|
| `require` 其他功能模块文件 | 产生隐式依赖，破坏按需加载 | 公共逻辑抽到 `MaBox_Helpers` |
| 直接调用其他模块的类方法 | 模块关闭时会报「类未定义」 | 通过 `MaBox_Helpers` 间接调用 |
| 在文件顶层执行逻辑 | 文件被加载时即执行，无法控制 | 所有逻辑放入 `activate()` 或回调方法 |
| 全局变量污染 | 模块间状态互相干扰 | 使用 `self::$option` 存储配置 |

### 3.4 公共 Helper 规范

创建 `includes/class-magick-helpers.php`，包含**所有模块都可能用到**的公共逻辑：

```php
<?php
/**
 * 公共工具类
 * 
 * 此类在插件初始化时必然加载，所有功能模块可安全调用其方法。
 */

if (!class_exists('MaBox_Helpers')) {
    class MaBox_Helpers
    {
        /**
         * 获取用户真实 IP
         */
        public static function get_real_ip()
        {
            $ip = '';
            if (!empty($_SERVER['HTTP_X_FORWARDED_FOR'])) {
                $ips = explode(',', $_SERVER['HTTP_X_FORWARDED_FOR']);
                $ip = trim($ips[0]);
            } elseif (!empty($_SERVER['HTTP_CLIENT_IP'])) {
                $ip = $_SERVER['HTTP_CLIENT_IP'];
            } else {
                $ip = $_SERVER['REMOTE_ADDR'] ?? '';
            }
            return sanitize_text_field($ip);
        }

        /**
         * 判断是否为移动端
         */
        public static function is_mobile()
        {
            return wp_is_mobile();
        }

        /**
         * 安全获取配置值
         */
        public static function get_config($config, $key, $default = false)
        {
            return MaBox_Admin::get_config($config, $key, $default);
        }

        /**
         * 判断当前用户是否已登录
         */
        public static function is_logged_in()
        {
            return is_user_logged_in();
        }

        /**
         * 获取当前文章 ID（在循环内有效）
         */
        public static function get_current_post_id()
        {
            return get_the_ID();
        }
    }
}
```

**加载顺序**：在 `class-magick-mixtrue.php` 的 `load_dependencies()` 中，`MaBox_Helpers` 必须第一个被加载：

```php
private function load_dependencies()
{
    // 1. 公共工具类（必然加载）
    require_once plugin_dir_path(dirname(__FILE__)) . 'includes/class-magick-helpers.php';
    require_once plugin_dir_path(dirname(__FILE__)) . 'includes/class-magick-mixtrue-tool.php';
    
    // 2. 管理类
    require_once plugin_dir_path(dirname(__FILE__)) . 'admin/class-magick-mixtrue-admin.php';
    
    // 3. 公共类
    require_once plugin_dir_path(dirname(__FILE__)) . 'public/class-magick-mixtrue-public.php';
}
```

改造 `MaBox_Admin::load()` 方法：

```php
public function load()
{
    // 获取选项
    $option = get_option(MAGICK_MIXTURE_OPTION);
    if (empty($option)) {
        return;
    }
    
    // 获取激活模块路由表
    $active_modules = get_option('Magick_ToolBox_Acttive_Modules', array());
    
    // 按需加载已激活的模块
    foreach ($active_modules as $module_path) {
        $file = plugin_dir_path(__FILE__) . 'partials/' . $module_path;
        if (file_exists($file)) {
            require_once $file;
            
            // 根据模块路径推断类名并激活
            $class_name = $this->resolve_class_name($module_path);
            if (class_exists($class_name)) {
                $config = $this->extract_module_config($option, $module_path);
                call_user_func(array($class_name, 'activate'), $config);
            }
        }
    }
}
```

## 五、静态资源管理规范

### 5.1 资源分级策略

| 级别 | 大小范围 | 加载方式 | 适用场景 |
|---|---|---|---|
| **L1 内联** | < 2KB | `wp_add_inline_script()` / `<style>` 直接输出 | 复制弹窗、全站变灰 CSS、简单 JS |
| **L2 独立** | 2-20KB | `wp_enqueue_script()` + 独立文件 | 动态标题、点击特效、滚动条美化 |
| **L3 延迟** | > 20KB / 有第三方依赖 | 条件入队 + `defer`/`async` | 页底鱼群（Canvas 库）、OwO 表情 |

### 5.2 资源入队规范

```php
<?php
/**
 * 静态资源加载示例
 */

class Npcink_Page_Copy_Pop_Up
{
    public static function activate($config)
    {
        $type = MaBox_Admin::get_config($config, 'copy_pop_up', 'false');
        if ($type === 'false') {
            return;
        }
        
        add_action('wp_footer', array(__CLASS__, 'render'));
    }
    
    public static function render()
    {
        $type = MaBox_Admin::get_config(self::$option, 'copy_pop_up');
        
        // L1：极轻量，内联输出
        if ($type === 'native') {
            echo '<script>
                document.addEventListener("copy", function() {
                    alert("复制成功！");
                });
            </script>';
            return;
        }
        
        // L2：中等体量，独立文件入队
        if ($type === 'sweetalert') {
            wp_enqueue_script(
                'mabox-copy-popup',
                plugin_dir_url(__DIR__) . 'public/dist/copy-popup.js',
                array(),
                MAGICK_MIXTURE_VERSION,
                true
            );
            wp_add_inline_script(
                'mabox-copy-popup',
                'CopyPopup.init(' . json_encode(self::$option) . ');'
            );
        }
    }
}
```

## 六、条件加载守卫规范

### 6.1 守卫类型

| 守卫类型 | 触发条件 | 示例 |
|---|---|---|
| **主题守卫** | 特定主题已激活 | B2 主题专属功能 |
| **页面守卫** | 仅在特定页面加载 | 首页特效、文章页功能 |
| **用户守卫** | 基于用户登录态 | 未登录隐藏内容 |
| **设备守卫** | 基于设备类型 | 移动端不加载某些特效 |
| **短代码守卫** | 页面包含特定短代码 | 足迹地图短代码 |

### 6.2 守卫实现模式

```php
<?php
/**
 * 条件加载守卫示例
 */

// 主题守卫
if (MaBox_Tool::theme_active('B2')) {
    require_once plugin_dir_path(__FILE__) . 'b2/index.php';
    Npcink_B2_Shop::activate($b2_config);
}

// 页面守卫
if (is_front_page()) {
    // 仅首页加载
    Npcink_Page_Home_Effect::activate($config);
}

// 用户守卫
if (!is_user_logged_in()) {
    Npcink_Page_Guest_Content::activate($config);
}

// 设备守卫
if (!wp_is_mobile()) {
    Npcink_Page_Desktop_Effect::activate($config);
}

// 短代码守卫（延迟注册）
add_shortcode('mabox_merc_map', function($atts) {
    // 仅在短代码被使用时才加载资源
```

## 九、错误处理与降级

### 9.1 模块加载失败处理

```php
foreach ($active_modules as $module_path) {
    $file = plugin_dir_path(__FILE__) . 'partials/' . $module_path;
    if (file_exists($file)) {
        try {
            require_once $file;
            $class_name = $this->resolve_class_name($module_path);
            if (class_exists($class_name)) {
                $config = $this->extract_module_config($option, $module_path);
                call_user_func(array($class_name, 'activate'), $config);
            }
        } catch (\Throwable $e) {
            error_log(sprintf(
                '[MaBox] Failed to activate module: %s | Error: %s',
                $module_path,
                $e->getMessage()
            ));
            // 从路由表中移除故障模块
            $active_modules = array_diff($active_modules, [$module_path]);
            update_option('Magick_ToolBox_Acttive_Modules', $active_modules);
        }
    }
}
```

```php
public function load()
{
    $option = get_option(MAGICK_MIXTURE_OPTION);
    if (empty($option)) {
        return;
    }
    
    $active_modules = get_option('Magick_ToolBox_Acttive_Modules', array());
    
    // 降级：如果路由表为空，回退到传统加载模式
    if (empty($active_modules)) {
        $this->load_legacy($option);
        return;
    }
    
    // 正常按需加载流程

}
```
