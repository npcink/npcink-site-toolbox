# 构建与发布指南

## 本地开发

### 前置要求

- Node.js >= 18.12.0
- pnpm（版本由 `vite/package.json` 的 `packageManager` 固定）
- PHP >= 7.4
- Composer

### 安装依赖

```bash
# PHP 依赖
composer install

# 前端依赖
corepack enable
cd vite
pnpm install --frozen-lockfile
```

### 开发模式

```bash
cd vite

# 启动管理面板开发服务器
pnpm run dev:admin

# 启动计数器开发服务器
pnpm run dev:count

# 启动公共前端开发服务器
pnpm run dev:public
```

### 构建前端资源

```bash
cd vite

# 构建所有项目
pnpm run build

# 或单独构建
pnpm run build:admin
pnpm run build:count
pnpm run build:public
```

## 发布流程

### CI 自动构建

推送代码到 `main` 或 `develop` 分支时，GitHub Actions 会自动：

1. 运行 PHP 语法检查（PHP 7.4 - 8.3）
2. 运行 PHPUnit 测试
3. 运行 PHPStan 静态分析
4. 运行 TypeScript 类型检查和 ESLint
5. 运行 Vitest 测试
6. 构建前端资源并打包插件 ZIP

### 手动发布

1. 确保所有代码已提交并推送
2. 在 GitHub Releases 中创建新版本
3. CI 会自动生成 `wp-magick-toolbox.zip` 构建产物
4. 下载 ZIP 文件并发布到 WordPress.org 或其他渠道

### 发布包内容

发布包包含：
- 所有 PHP 文件（admin/, includes/, public/）
- 已构建的前端资源（vite/*/dist/）
- 插件主文件（magick-tool-box.php）
- 必要资源文件（css/, admin/partials/*/css/ 等）

发布包排除：
- 源代码（vite/*/src/）
- 配置文件（package.json, vite.config.* 等）
- 开发工具（tests/, docs/, ai/ 等）
- 依赖目录（node_modules/, vendor/）

## 目录结构

```
wp-magick-toolbox/
├── admin/                    # 后台 PHP 代码
│   ├── class-magick-mixture-admin.php
│   ├── modules/              # 模块加载器
│   └── partials/             # 模块实现
├── includes/                 # 核心类
│   ├── class-magick-mixture.php
│   ├── class-magick-config-manager.php
│   └── ...
├── public/                   # 前台 PHP 代码
├── vite/                     # 前端源码（开发用）
│   ├── admin/                # 管理面板 React 应用
│   │   ├── src/              # 源码
│   │   └── dist/             # 构建产物（发布用）
│   ├── count/                # 计数器应用
│   └── public/               # 公共前端
├── css/                      # 公共 CSS
└── magick-tool-box.php       # 插件主文件
```

## 注意事项

1. **不要提交 dist 目录**：`vite/*/dist/` 已在 `.gitignore` 中排除
2. **发布前确保构建**：CI 会自动构建，本地测试需手动运行 `pnpm run build`
3. **版本号更新**：发布新版本时更新 `magick-tool-box.php` 中的 `Version` 和 `MAGICK_MIXTURE_VERSION` 常量
