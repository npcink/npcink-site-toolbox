# Npcink Site Toolbox 3.2.0 BUG 排查与验证台账

> 日期：2026-07-20
> 状态：当前开发阶段 T1-T8 与真实使用工作包 R1-R2 已完成；T9-T11 临近正式发布时执行
> 范围：3.2.0 功能开发阶段的真实 WordPress 行为验证；不构建发布包

## 一、目的与边界

本台账用于记录 3.2.0 在正式发布前仍值得执行的 BUG 排查任务、完成目标和实机证据。执行原则是先验证真实缺口，只修复可复现的行为问题，不把技术债、兼容层或新功能混入排查工作。

本轮包含：

- 五个编辑器工具的保存、重载和前台预览闭环；
- GitHub 公共 API 的失败降级与缓存；
- OSS 敏感字段映射和当前 REST 路由的权限、输入边界；
- 数据库清理、WP-Cron、媒体上传和高频模块的真实 WordPress 运行路径；
- 临近发布前的停用、卸载、最低版本和主题兼容冒烟。

本轮不包含：

- 新增区块、设置页、兼容层、AI Runtime 或第三个 Vite 构建目标；
- 全仓 React 警告清零、`any` 清理或 Plugin Check 已确认误报重构；
- 未经明确决定的 Multisite、所有主题和所有第三方插件组合；
- `composer release:build`、发布 ZIP、WordPress.org/SVN 或正式发布操作。

## 四、执行顺序与停止条件

1. 每次只让一个工作包拥有一个运行模块；可并行的任务必须使用互不重叠的代码与测试文件区域。
2. 先记录基线，再创建带 `npcink_rc320_` 前缀的测试内容、Option、Transient 或媒体夹具。
3. 只修复能够稳定复现、影响实际行为的 BUG；纯样式偏好、历史警告和工具误报记录后延后。
4. 一旦涉及删除或卸载，先确认 Local 路径和站点 URL，再导出数据库；不在用户现有内容上试删。
5. 每项完成后恢复设置、文章、附件、Transient、Cron 和测试用户，并复查 PHP、PHP-FPM、Nginx 日志及浏览器控制台。
6. 若出现真实 BUG，先补能在旧实现失败的聚焦回归，再做最小修复；聚焦门禁通过后才进入下一任务。

## 十三、R4：OSS 上传目录与阿里云 Endpoint 输入契约

- 日期：2026-07-20。本节取代 R3 中“阿里云填写 Region”的配置指引；R3 的连接测试、权限和隐私结论继续有效。

- 决定：`performance.oss` 增加非敏感 `path` 与 `endpoint`。Bucket 只保存空间名称，`path` 保存可选对象键前缀；阿里云直接填写控制台外网 Endpoint，也接受 `cn-shanghai` 快捷写法。腾讯云继续填写 `ap-beijing` 形式的 Region；七牛云不展示无效地域字段。公开访问地址独立保存，连接测试不再要求它先配置。
