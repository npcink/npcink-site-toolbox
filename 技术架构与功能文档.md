## 1. 项目概述

### 1.1 产品定位

WP Magick Toolbox 是一款面向中国 WordPress 站长的免费工具箱插件，集成了 **90+ 实用功能**，涵盖站点优化、SEO、安全防护、国内生态对接（百度推送、微信生态、备案合规）、性能优化等多个维度。

**核心理念**: 一个插件，解决站长 80% 的日常需求。

## 2. 技术架构

### 2.1 后端架构

**技术栈**: PHP 7.4+, WordPress Plugin Boilerplate 变体

**核心类职责**:

| 类名 | 文件 | 职责 |
|------|------|------|
| `Magick_Mixtrue` | `includes/class-magick-mixtrue.php` | 主插件类，负责加载依赖、注册钩子、启动插件 |
| `MaBox_Admin` | `admin/class-magick-mixtrue-admin.php` | 后台管理，菜单注册、资源加载、REST API、AJAX 处理 |
| `MaBox_Public` | `public/class-magick-mixtrue-public.php` | 前台公共功能 |
| `MaBox_Config_Manager` | `includes/class-magick-config-manager.php` | 配置管理（读取/保存/迁移/导出/导入/缓存） |
| `MaBox_Module_Loader` | `admin/modules/loader.php` | 模块动态加载器 |
| `MaBox_Tool` | `includes/class-magick-mixtrue-tool.php` | 通用工具函数 |
| `Magick_Helpers` | `includes/class-magick-helpers.php` | 辅助函数集合 |

**执行流程**:

```
magick-tool-box.php
  └─ run_magick_mixture()
       └─ Magick_Mixtrue::__construct()
            ├─ load_dependencies()     # 加载核心类
            ├─ define_admin_hooks()    # 注册后台钩子
            └─ define_public_hooks()   # 注册前台钩子
       └─ Magick_Mixtrue::run()
            └─ script_loader_tag 过滤器 (添加 type="module")
```

### 2.2 前端架构

**技术栈**: React + TypeScript + Vite + Ant Design + TailwindCSS

**三个独立项目**:

| 项目 | 路径 | 用途 |
|------|------|------|
| admin | `vite/admin/` | 后台设置界面，包含所有功能开关和配置表单 |
| count | `vite/count/` | 图表展示组件（ECharts + 自研统计） |
| public | `vite/public/` | 前端展示组件（按需加载） |

**数据流**:

```
PHP (wp_localize_script)
  └─ window.dataLocal
       ├─ option       # 配置数据 (MaBox_Config_Manager::get_merged_config())
       ├─ cat_arr      # 分类数据
       ├─ single_arr   # 文章数据
       ├─ url_site     # 站点 URL
       ├─ ajaxurl      # AJAX 地址
       └─ nonce        # CSRF 令牌

React (DataContext)
  └─ 读取配置 → 表单绑定 → 保存时 POST 到 REST API
```

### 2.3 构建与部署

- **开发**: 在 `vite/` 下执行 `pnpm dev:admin` / `pnpm dev:count` / `pnpm dev:public`
- **构建**: 在 `vite/` 下执行 `pnpm build` (统一生成各项目的 `dist/` 目录)
- **部署**: 仅保留各 Vite 项目中 `dist/` 目录下的文件

## 3. 核心模块系统

### 3.1 模块注册表 (`admin/modules/registry.php`)

采用声明式注册表替代硬编码加载，每个模块定义以下元数据:

| 字段 | 类型 | 说明 |
|------|------|------|
| `class` | string | 模块类名 |
| `file` | string | 模块文件路径 (相对于 `admin/partials/`) |
| `option_key` | string | 配置键名 (用于判断是否激活) |
| `category` | string | 模块分类 (optimize/page/function 等) |
| `scope` | string | 作用域: `admin`/`frontend`/`both` |
| `config_path` | string | (可选) 传递给模块的配置路径 |
| `always_load` | bool | (可选) 是否始终加载 |
| `mobile_only` | bool | (可选) 仅移动端加载 |
| `theme_requirement` | string | (可选) 依赖的主题名称 |
| `risk_tags` | array | (可选) 风险标签 (推荐/谨慎/安全/SEO/性能 等) |

### 3.2 模块加载器 (`admin/modules/loader.php`)

**核心方法**:

| 方法 | 说明 |
|------|------|
| `get_registry()` | 加载并缓存注册表 |
| `get_active_modules($config)` | 根据配置计算激活的模块列表 |
| `load_module($module_id, $config)` | 动态加载单个模块 |
| `get_module_meta($module_id)` | 获取模块元数据 |
| `get_modules_by_category($category)` | 按分类获取模块 |
| `get_all_module_ids()` | 获取所有模块 ID |

**按需加载机制**:

1. 用户保存配置时，`MaBox_Module_Loader::get_active_modules()` 计算激活模块
2. 结果存入 `Magick_ToolBox_Active_Modules` option
3. 同时写入对象缓存 (1 小时 TTL)
4. 页面加载时从缓存/数据库读取路由表
5. 仅加载激活的模块，跳过未启用的功能

**Scope 过滤**:
- `admin`: 仅后台请求加载
- `frontend`: 仅前台请求加载
- `both`: 前后都加载

### 3.3 模块接口契约 (`includes/interface-mabox-module.php`)

所有模块应实现 `MaBox_Module_Interface` 接口，确保统一的 `run($config)` 方法签名。

**标准模块结构**:

```php
if (!class_exists('MaBox_Module_Name')) {
    class MaBox_Module_Name implements MaBox_Module_Interface {
        private static $option;
        
        public static function run($config) {
            self::$option = $config;
            // 注册钩子、添加过滤器等
            add_action('wp_footer', array(__CLASS__, 'display'));
        }
        
        public static function display() {
            $val = MaBox_Admin::get_config(self::$option, 'feature_key');
            if ($val === true) {
                // 功能实现
            }
        }
    }
}
```

## 4. 数据存储与配置管理

### 4.1 配置存储策略

**按模块拆分为多个 `wp_options` 键** (自 v2.1.0):

| Option 键名 | 用途 |
|------------|------|
| `Magick_ToolBox_Option_Optimize` | 站点优化配置 |
| `Magick_ToolBox_Option_Page` | 页面外观/功能配置 |
| `Magick_ToolBox_Option_Function` | 辅助功能/SEO 配置 |
| `Magick_ToolBox_Option_H5` | H5 移动端配置 |
| `Magick_ToolBox_Option_Login` | 登录安全配置 |
| `Magick_ToolBox_Option_Shortcode` | 短代码配置 |
| `Magick_ToolBox_Option_Template` | 页面模板配置 |
| `Magick_ToolBox_Option_Domestic` | 国内生态配置 |
| `Magick_ToolBox_Option_Performance` | 性能优化配置 |
| `Magick_ToolBox_Option_Services` | 增值服务配置 |
| `Magick_ToolBox_Option_Feedback` | 用户反馈配置 |

### 4.2 配置管理器 (`MaBox_Config_Manager`)

**核心功能**:

| 方法 | 说明 |
|------|------|
| `get_merged_config()` | 合并所有模块配置 (带缓存) |
| `save_full_config($data)` | 保存完整配置 (按模块拆分存储) |
| `export_config()` | 导出所有配置为数组 |
| `import_config($data)` | 从数组导入配置 |
| `migrate()` | 配置迁移 (从旧版单 option 到拆分 option) |
| `clear_cache()` | 清除配置缓存 |

### 4.3 备份与迁移

- **自动快照**: 最多保留 5 个配置快照
- **迁移标记**: `Magick_ToolBox_Config_Version` 记录当前配置版本
- **回滚机制**: 保存失败时自动恢复备份
- **备份键**: `Magick_ToolBox_Option_Backup_v210`

### 4.4 路由表缓存

- **路由表 option**: `Magick_ToolBox_Active_Modules` (数组，存储激活的模块 ID)
- **对象缓存**: `wp_cache_get/set('mabox_active_modules', 'mabox')` (1 小时 TTL)
- **降级策略**: 缓存未命中时从数据库读取并重建

## 7. 安全机制

### 7.1 SQL 注入防护

- 所有 SQL 查询使用 `$wpdb->prepare()`
- 参数化查询，禁止字符串拼接

### 7.2 CSRF 防护

- REST 端点: WordPress 内置 nonce 验证
- AJAX 端点: `check_ajax_referer('mabox_save_nonce', 'nonce')`
- 公开 API: 自定义 `x-mabox-nonce` header 验证

### 7.3 XSS 防护

- 输出转义: `esc_html()` / `esc_url()` / `esc_attr()`
- JavaScript 上下文: `wp_json_encode()` / `esc_js()`

### 7.4 权限检查

- 管理操作: `current_user_can('manage_options')`
- 内容编辑: `current_user_can('edit_posts')`
- 公开接口: `__return_true` + nonce 验证

### 7.5 输入清洗

- `sanitize_text_field()` 处理文本输入
- `wp_unslash()` 处理 POST 数据
- `json_decode()` 验证 JSON 格式

### 7.6 保存回滚机制

```php
$old_option_backup = MaBox_Config_Manager::get_merged_config();
$result = MaBox_Config_Manager::save_full_config($data);
if (!$result['success']) {
    MaBox_Config_Manager::save_full_config($old_option_backup);
    // 返回错误
}
```

## 9. 开发规范

### 9.1 后端 PHP 开发规范

**位置**: `admin/partials/[category]/[feature_name].php`

**结构要求**:
- 使用静态类
- 必须包含 `run($config)` 方法
- 使用 `class_exists` 检查避免重复加载

**示例**:
```php
if (!class_exists('MaBox_Module_Name')) {
    class MaBox_Module_Name implements MaBox_Module_Interface {
        private static $option;
        
        public static function run($config) {
            self::$option = $config;
            add_action('wp_footer', array(__CLASS__, 'display'));
        }
        
        public static function display() {
            $val = MaBox_Admin::get_config(self::$option, 'feature_key');
            if ($val === true) {
                echo 'Hello World';
            }
        }
    }
}
```

**注册步骤**:
1. 在 `admin/modules/registry.php` 中添加模块记录
2. 确保 `option_key` 与前端配置键匹配
3. 设置正确的 `scope` (admin/frontend/both)

### 9.2 前端 React 开发规范

**类型定义**: `vite/admin/src/tool/interface.tsx` 中添加字段  
**默认值**: `vite/admin/src/tool/defaultVar.tsx` 中添加默认值  
**UI 组件**: 在对应 Tab 组件中添加 Ant Design 表单控件  
**绑定**: `optionData.{module}.{feature_key}`

**统一开关**: 使用 `FeatureSwitch` 组件，自动接入收藏 + 风险检测

### 9.3 资源加载规范

- **CSS/JS**: 使用 `wp_enqueue_style` / `wp_enqueue_script`
- **内联脚本**: 使用 `wp_localize_script` 传递 PHP 变量到 JS
- **按需加载**: 功能未开启时不注册任何 Hook 或资源
- **Module 类型**: 通过 `script_loader_tag` 过滤器添加 `type="module"`

### 9.4 安全规范

- SQL: 全部使用 `$wpdb->prepare()`
- 输出: `esc_html()` / `esc_url()` / `esc_attr()`
- AJAX: `check_ajax_referer` + `current_user_can`
- 输入: `sanitize_text_field` / `wp_unslash`
