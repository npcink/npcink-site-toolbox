# Implementation Plan: v2.6 产品化稳定版

## Overview

v2.6 不堆功能，主攻"新手能配好、核心刚需稳定、问题能诊断"。基于代码库实际状态，5 个阶段、24 个任务，约 30 天。

## Architecture Decisions

- 保留现有配置模型（12 split options）和模块注册表（75 modules），不做架构迁移
- 新增配置分组 `domestic.environment`，复用 `MAGICK_MIXTURE_OPTION_DOMESTIC` option
- 新增 REST 端点均挂在 `mabox/v1` 命名空间，遵循现有 `manage_options` + nonce + schema + audit 模式
- 配置向导复用现有 `presets.ts` 深合并机制 + `diff-modal.tsx` 确认
- 高风险功能治理基于现有 `tiers.php` 扩展 + `riskyFeature.tsx` 弹窗增强
- 诊断报告导出已存在（Markdown 格式），v2.6 增加敏感字段脱敏

## Codebase Current State (Critical Findings)

| 项 | 状态 | 影响 |
|---|---|---|
| `baidu_push/index.php` L21 | 多余 `}` 导致 parse error | 百度推送完全不可用 |
| `search_enhance/index.php` L23 | 同模式 parse error | 搜索高亮/推荐不可用 |
| `compose/runcode/demo.php` L61 | 混合 PHP tag 导致 parse error | 短代码运行器不可用 |
| `single_show_id.php` L39-40 | `${var}` PHP 8.2+ 废弃警告 | 未来版本兼容风险 |
| `performance.db_clean` | 在 core 层级而非 high_risk | 与 v2.6 治理策略不一致 |
| `domestic.login_security` | IP 锁定 `get_client_ip()` 信任 `X-Forwarded-For` | IP 伪造绕过风险 |
| 现有 presets.ts | 7 个方案，场景不对齐 v2.6 规划 | 需新增 3 个向导方案 |
| Dashboard | 2 个评分卡重复（前端 + 后端诊断） | 需统一 |

## Risks and Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| 国内环境适配检测在海外服务器误判 | Low | 检测结果缓存 1 小时 + 手动刷新 |
| 向导方案不适用所有用户 | Med | 保留手动配置入口，向导可跳过 |
| 高风险功能重分类影响已有用户 | Med | 仅改 UI 展示，不改功能逻辑；已有配置不受影响 |
| IP 锁定修复影响 Nginx 反代用户 | High | 新增 trusted_proxies 配置项，文档引导配置 |
| 诊断报告脱敏遗漏字段 | Med | 脱敏字段列表与 config schema 对齐，代码审查 |
