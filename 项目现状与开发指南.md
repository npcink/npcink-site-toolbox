## 一、 技术架构

### 1. 后端 (PHP)

- **版本**: PHP 7.4+
- **架构**: WordPress Plugin Boilerplate 变体 + 模块注册表机制
- **数据存储**: 按模块拆分为多个 `wp_options` 键；浏览器端配置快照已在 Pre-GA 重构中删除
- **核心类**: `MaBox_Admin`（位于 `admin/class-magick-mixtrue-admin.php`）
  - 负责注册菜单、加载资源、处理 REST API 请求
  - 提供 `get_config($config, $key, $default)` 方法用于读取配置（支持数组和对象）
  - 提供 `get_seting($key)` 方法用于获取顶层配置
- **模块加载**: `admin/modules/registry.php`（注册表）+ `admin/modules/loader.php`（统一加载器）

### 2. 前端 (React)

- **技术栈**: React + TypeScript + Vite + Ant Design + TailwindCSS
- **构建目录**: `vite/admin/`（设置页）、`vite/count/`（图表）、`vite/public/`（前端展示）
- **数据流**:
  - PHP 只通过 `wp_localize_script` 注入 REST 基址、nonce 和非配置启动数据
  - React 通过 `GET /mabox/v1/settings` 读取非敏感设置与凭据状态
  - 保存时通过 `POST /mabox/v1/settings` 提交普通设置和显式凭据变更
- **类型定义**: `vite/admin/src/tool/interface.tsx`
- **默认值**: `vite/admin/src/tool/defaultVar.tsx`

### 3. 安全现状

- **SQL 注入**: 所有 SQL 查询已改用 `$wpdb->prepare()`
- **CSRF**: REST 端点使用 nonce 验证，AJAX 端点使用 `check_ajax_referer`
- **XSS**: 所有动态输出已使用 `esc_html()` / `esc_url()` / `esc_attr()` 转义
- **权限**: 敏感操作已添加 `current_user_can('manage_options')` 检查
- **REST API**: 16+ 端点已统一注册，admin-ajax 保留为兼容层（已标记弃用）

## 三、 开发规范（如何添加新功能）

### 1. 后端 PHP 开发规范

- **位置**: `admin/partials/[category]/[feature_name].php`
- **结构**: 必须使用静态类，包含 `run($config)` 方法
- **示例**:
  ```php
  if (!class_exists('Npcink_Page_New_Feature')) {
      class Npcink_Page_New_Feature {
          private static $option;
          public static function run($config) {
              self::$option = $config;
              add_action('wp_footer', array(__CLASS__, 'display'));
          }
          public static function display() {
              $val = MaBox_Admin::get_config(self::$option, 'new_feature_key');
              if ($val === true) { echo 'Hello World'; }
          }
      }
  }
  ```
- **注册**: 在 `admin/modules/registry.php` 中添加模块记录
- **安全**: SQL 用 `$wpdb->prepare()`、输出用 `esc_html()` / `esc_url()`、AJAX 用 `check_ajax_referer` + `current_user_can`

### 2. 前端 React 开发规范

- **类型定义**: `vite/admin/src/tool/interface.tsx` 中添加字段
- **默认值**: `vite/admin/src/tool/defaultVar.tsx` 中添加默认值
- **UI 组件**: 在对应 Tab 组件中添加 Ant Design 表单控件，绑定 `optionData.{module}.{feature_key}`
- **统一开关**: 使用 `FeatureSwitch` 组件（位于 `basic/feature-switch.tsx`），自动接入收藏 + 风险检测

### 3. 资源加载规范

- **CSS/JS**: 使用 `wp_enqueue_style` / `wp_enqueue_script`
- **内联脚本**: 使用 `wp_localize_script` 传递 PHP 变量到 JS
- **按需加载**: 功能未开启时不注册任何 Hook 或资源
