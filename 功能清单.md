### 统计口径与 Registry 差异

- 上表按用户可理解的功能或任务入口计数；同一入口中的配置字段和开关不再逐项计数，导航、搜索、保存与帮助等管理外壳也不计入。
- 当前 Registry 共 55 个运行时条目：`optimize` 18 个、`page` 18 个、`function` 10 个、`domestic` 4 个、`performance` 5 个。
- `page.interface_category_data` 是分类数据接口提供者，不是独立用户功能，因此从 Registry 口径减 1。
- `page.hide_category`、`page.hide_tag`、`page.hide_page` 在表中合并为“隐藏指定内容”，相对 Registry 口径再减 2。
- `domestic.compliance` 在表中拆为备案、Cookie、版权 3 项，相对 Registry 口径加 2；`domestic.wechat` 和 `domestic.login_security` 各拆为 2 项，各加 1。
- “站点体检中心”和“中国访问适配”是由诊断/REST 服务提供的用户任务，不是 Registry 运行模块，各加 1。
- 3 个区块样板和 2 个动态区块属于编辑器能力，不进入设置 Registry，另加 5。
- 因此用户可见条目合计为 `55 - 1 - 2 + 2 + 1 + 1 + 1 + 1 + 5 = 63`，与 Registry 的 55 个模块并不矛盾。

## 技术架构

- **后端**：PHP 7.4+、WordPress 6.3+，WordPress 插件标准架构
- **前端设置页**：React + TypeScript + Vite + Ant Design
- **数据存储**：WordPress `wp_options` 表
- **通信方式**：WordPress REST API

## 安全加固（已完成）

- SQL 注入防护（全部使用 `$wpdb->prepare()`）
- CSRF 防护（nonce 验证）
- XSS 防护（输出转义）
- 权限检查（`current_user_can`）
- 输入清洗（`sanitize_text_field`）
