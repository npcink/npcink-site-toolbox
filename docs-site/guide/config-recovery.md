# 配置恢复

v2.5 提供了更灵活的配置恢复能力：既可以**全量恢复默认配置**，也可以**按模块恢复默认值**。

## 全量恢复默认配置

如果你希望一次性将所有设置还原到插件初始状态：

1. 进入后台首页（仪表盘）
2. 点击**配置备份中心**按钮
3. 在弹出的 Modal 中点击红色的**恢复默认配置**按钮
4. 确认后，所有模块设置将被重置为默认值
5. **必须点击保存按钮**才能将变更写入服务器

::: warning 注意
全量恢复会覆盖你所有的自定义配置，操作前建议先导出备份或创建快照。
:::

## 单模块恢复默认值

如果你只想重置某个模块，而保留其他模块的现有配置：

1. 进入后台首页（仪表盘）
2. 在**一键配置方案**卡片中，点击**恢复模块默认值**下拉按钮
3. 选择要恢复的模块：
   - 站点优化（optimize）
   - 页面功能（page）
   - SEO / 功能（function）
   - 登录页（login）
   - 国内生态（domestic）
   - 性能优化（performance）
   - AI 审核（ai_review）
   - 增值服务（services）
   - 用户反馈（feedback）
4. 确认后，该模块下的所有子设置将被重置为默认值
5. **必须点击保存按钮**才能将变更写入服务器

## 配置备份中心

配置备份中心提供以下能力：

### 自动快照

每次点击**保存**按钮时，系统会自动在本地浏览器中保存一份配置快照。最多保留最近 5 次。

- **恢复**：选择某个历史快照，点击"恢复"，当前配置将被替换为快照内容
- **删除**：删除不需要的历史快照
- **清除所有**：一键清空所有快照

### 快照与恢复的区别

| 能力 | 快照 | 恢复默认 |
|------|------|----------|
| 数据来源 | 你之前保存的配置 | 插件内置默认值 |
| 覆盖范围 | 全量替换 | 全量或单模块 |
| 持久化 | 仅本地浏览器 | 服务端 Schema |
| 用途 | 回滚到历史状态 | 重置为出厂设置 |

## 保存确认机制

v2.5 引入了保存前 diff 弹窗：

1. 点击**保存**按钮时，系统会先对比当前配置与初始配置
2. 如果没有任何更改，提示"无需保存"
3. 如果有更改，弹出变更列表，展示：
   - 变更项名称
   - 变更前后的值
   - 高风险项标红警示
4. 确认后才会真正提交到服务器

这样可以避免误操作，尤其在开启高风险功能时给你二次确认的机会。
