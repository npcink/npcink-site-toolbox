# 常见问题

## 安装与使用

### 支持哪些版本？

最低要求为 WordPress 4.6 和 PHP 7.4。建议使用仍获官方安全维护的 WordPress 与 PHP 版本。

### 如何找到特定功能？

使用设置页顶部搜索框；结果可直接定位到对应语义分区和设置项。

### 开启功能后站点异常怎么办？

关闭刚刚变更的功能并保存，然后清理缓存。若后台无法打开，可先在 WordPress 插件页停用插件并检查 PHP 错误日志。站点级恢复应使用主机或数据库备份。

## 配置管理

### 设置存在哪里？

设置按模块存储在 WordPress `wp_options` 中。实际 option 映射由 `MaBox_Config_Manager` 管理，不应依赖旧文档中的固定键清单。

### 为什么不能导入或导出设置 JSON？

完整配置可能包含凭据。为避免凭据进入浏览器快照或下载文件，Pre-GA 版本删除了设置导入导出和本地快照。跨站迁移请使用受控的数据库迁移流程。

### 为什么密钥输入框是空的？

这是预期行为。页面只知道凭据是否已配置，不会读取原值。不操作会保留已有值；只有显式替换或清除并保存才会改变服务端凭据。

### 设置加载失败后为什么不能保存？

这是防覆盖保护。刷新页面并排查 REST/nonce/权限问题，读取恢复前不要尝试用默认值重建配置。

## 兼容性

### 与 SEO 插件冲突怎么办？

避免同时启用两个插件对同一页面输出 TDK。保留一方作为单一事实源，并清理缓存后核对最终 HTML。

### 与缓存插件兼容吗？

大多数情况下兼容。修改前台相关配置后应清理页面缓存；维护页和评论功能需要按缓存插件文档设置排除规则。

## 开发

新增或修改功能前请阅读仓库 `AGENTS.md` 与 Pre-GA ADR，并同步更新 Schema、模块注册、前端类型和自动化测试。
